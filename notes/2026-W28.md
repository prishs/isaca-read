## CISO Series / Defense in Depth — Humans Are Bottleneck in a Machine-Speed World (July 9, 2026)
- Panelists (David Spark, Howard Holton, Adam Glick, with input from Rinki Sethi, CSO at Upwind Security) argued that "human-in-the-loop" review is becoming a throughput bottleneck as security tooling moves toward machine-speed decisioning — the math of requiring human sign-off doesn't scale to agentic-attack timelines.
- The panel debated when automated systems should be trusted to act on security decisions without human validation, framing it as a graduated trust question rather than all-or-nothing autonomy.
- They mapped an evolution path for security tools from "assistant" (suggests, human decides) to "autonomous" (acts, human audits after), and argued security roles need to shift from operators to overseers/tuners of these systems as that shift happens.

[Episode link](https://defenseindepth.libsyn.com/humans-are-bottleneck-in-a-machine-speed-world)

## Cloud Security Podcast — The Hidden Cost of BlackBox AI: Bridging Cloud and Code Security (July 9, 2026)
- The hosts distinguished vulnerability "reachability" (is the code path live) from true "exploitability" (can an attacker actually trigger it), arguing AI-assisted triage only pays off once tools reliably separate the two — legacy SCA/SAST tools produce roughly nine false positives for every real finding.
- A Fortune 100 case study showed an in-house AI security build hitting an estimated $4M/week in token costs when extrapolated across the environment, making cost-per-finding as important a vendor-evaluation criterion as accuracy.
- Buyers were told to treat "black box" AI security vendors as a red flag: probe for what a vendor adds beyond a wrapped foundation-model API, since LLMs are increasingly acting as translators that let AppSec and cloud security findings be correlated in one "security brain" rather than two siloed tool sets.

[Episode link](https://www.cloudsecuritypodcast.tv/videos/the-hidden-cost-of-blackbox-ai-bridging-cloud-and-code-security)

## ISACA Podcast — Why You Should Use the F Word More (July 6, 2026)
- Host Safia Kazi and guest Jake Bernardes (CISO, Anecdotes) unpacked FedRAMP 20x, the federal government's overhauled continuous-authorization approach, and why its shift toward automation-driven evidence changes what "audit-ready" means for cloud vendors serving federal customers.
- Bernardes argued FedRAMP 20x will push GRC programs toward continuous monitoring and machine-readable, "audit-grade" data rather than point-in-time assessments, reshaping how compliance evidence is collected and reviewed.
- The discussion framed this as a preview of where private-sector compliance is headed too: enterprise customers are starting to expect the same continuous, automated evidence trail from vendors that federal agencies will require under FedRAMP 20x.

[Episode link](https://www.isaca.org/resources/news-and-trends/isaca-podcast-library/isaca-podcast-why-you-should-use-the-f-word-more)

## Audit, CISO & VAPT angle
The FedRAMP 20x conversation is the most actionable item for audit clients this week: any vendor serving federal customers should be asked whether their compliance evidence pipeline is moving toward continuous, automated collection, since that's likely to become the private-sector baseline too. The Cloud Security Podcast's reachability-vs-exploitability framing and cost-of-AI-triage data point are worth bringing into VAPT scoping discussions — clients evaluating AI-assisted AppSec/cloud tools need pricing and false-positive-rate commitments up front, not just capability demos. The Defense in Depth debate on human-in-the-loop bottlenecks belongs in board-level conversations about where autonomous security tooling is being granted decision authority, since that's a governance and accountability question as much as a technical one.
