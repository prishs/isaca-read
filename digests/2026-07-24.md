# Security & Audit Digest — 2026-07-24

## Top items

### 1. Russian State-Sponsored Actors Exploited Zimbra Zero-Day to Steal Mail and 2FA Codes
**Why it matters for audit/CISO/VAPT:** A federal advisory ties a nation-state group to months of undetected mailbox access via a webmail flaw, defeating MFA reliance — audit teams should verify patch status and run compromise assessments on any Zimbra deployment.
CISA and partners detailed a Laundry Bear (Void Blizzard) campaign that combined phishing with a now-patched Zimbra zero-day to exfiltrate 90 days of email, directory listings, and browser-saved credentials from Western government and commercial targets. ([source](https://www.cisa.gov/news-events/cybersecurity-advisories/aa26-204a))

### 2. Critical SharePoint RCE Under Active Exploitation via Public PoC — Federal Deadline Tomorrow
**Why it matters for audit/CISO/VAPT:** A public proof-of-concept has turned a critical SharePoint deserialization bug into a live threat, with the KEV-mandated federal remediation deadline landing 2026-07-25 — a hard checkpoint for patch-compliance audits.
watchTowr reported active in-the-wild exploitation of CVE-2026-50522 (CVSS 9.8) following release of a working exploit, affecting on-prem and hybrid SharePoint environments. ([source](https://thehackernews.com/2026/07/critical-sharepoint-rce-cve-2026-50522.html))

### 3. WordPress wp2shell RCE Chain Fuels Mass Scanning
**Why it matters for audit/CISO/VAPT:** Two chained, now-KEV-listed WordPress flaws enable trivial unauthenticated site takeover and are already driving mass scanning — a priority check for any organization running WordPress-based sites or plugins.
CVE-2026-63030 and CVE-2026-60137, when combined, allow unauthenticated remote code execution; both were added to CISA's KEV catalog on 2026-07-21. ([source](https://thehackernews.com/2026/07/wordpress-wp2shell-exploitation-grows.html))

### 4. Windmill Path Traversal Actively Exploited to Leak Server Secrets
**Why it matters for audit/CISO/VAPT:** Unauthenticated file disclosure on a developer-automation platform can leak credentials that feed downstream systems; VulnCheck confirms active exploitation, not just PoC risk.
CVE-2026-29059 (CVSS 7.5) is an unauthenticated path traversal in Windmill's log-retrieval endpoint, already exploited in the wild. ([source](https://thehackernews.com/2026/07/hackers-exploit-windmill-flaw-to-read.html))

### 5. Qilin Ransomware Exploits PAN-OS Authentication Bypass for Initial Access
**Why it matters for audit/CISO/VAPT:** A perimeter firewall flaw is now a confirmed ransomware entry point — VAPT scopes covering PAN-OS appliances should treat unpatched instances as an active breach path, not a theoretical risk.
Arctic Wolf Labs found Qilin (Agenda) affiliates using a now-patched PAN-OS authentication bypass to gain initial access in multiple June 2026 intrusions. ([source](https://thehackernews.com/2026/07/qilin-ransomware-attackers-exploit-pan.html))

### 6. Azure DevOps MCP Flaw Lets Hidden PR Comments Hijack AI Review Agents
**Why it matters for audit/CISO/VAPT:** A single invisible pull-request comment can redirect an AI coding agent into repos the attacker has no rights to and exfiltrate what it finds — a new prompt-injection class VAPT scopes should start testing for.
The flaw sits in Microsoft's official Azure DevOps MCP server and turns a reviewer's own AI agent against them without any visible trigger. ([source](https://thehackernews.com/2026/07/microsoft-azure-devops-mcp-flaw-lets.html))

### 7. Claude Cowork Sandbox Escape Could Expose Mac Files to AI Agent
**Why it matters for audit/CISO/VAPT:** Another agentic-AI containment failure, days after the OpenAI/Hugging Face incident, reinforces that VM/sandbox boundaries around AI coding tools cannot yet be treated as a reliable control.
Researchers at Accomplish AI found a sandbox escape in Anthropic's Claude Cowork letting the agent break out of its Linux VM to read or write arbitrary files on the host Mac. ([source](https://thehackernews.com/2026/07/claude-cowork-flaw-could-let-ai-agent.html))
