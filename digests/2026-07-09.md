# Security & Audit Digest — 2026-07-09

## Top items

### 1. BeyondTrust patches critical auth bypass flaws in Remote Support and PRA
**Why it matters for audit/CISO/VAPT:** BeyondTrust RS/PRA sit at the center of many privileged-access-management programs; an unauthenticated bypass in the very tools used to govern admin and vendor remote access is a direct hit to PAM assurance.
BeyondTrust shipped fixes for two critical flaws in Remote Support and Privileged Remote Access that could let unauthenticated attackers take control of affected systems. No confirmed in-the-wild exploitation yet, but patch priority should be high given the products' privileged footprint. ([The Hacker News](https://thehackernews.com/2026/07/beyondtrust-patches-critical-auth.html))

### 2. "GhostLock" — 15-year-old Linux kernel flaw enables root and container escape
**Why it matters for audit/CISO/VAPT:** Ships by default in nearly every mainstream Linux distribution and lets any logged-in user reach root, undermining the privilege-separation and container-isolation assumptions behind most audit control matrices.
Nebula Security disclosed GhostLock (CVE-2026-43499), a 15-year-old Linux kernel flaw that lets a local user escalate to full root and escape containers on unpatched systems. ([The Hacker News](https://thehackernews.com/2026/07/15-year-old-ghostlock-flaw-enables-root.html))

### 3. Ubiquiti patches critical UniFi flaws across Connect, Talk, Access, Protect, and OS
**Why it matters for audit/CISO/VAPT:** UniFi gear underpins access control, VoIP, and video surveillance for many mid-size enterprises; a max-severity command-injection bug across the whole line is an urgent patch-compliance and physical-security-control item.
Ubiquiti released updates for multiple critical flaws spanning UniFi Connect, Talk, Access, Protect, and OS, including a maximum-severity UniFi OS bug exploitable for command injection and other bugs enabling privilege escalation. ([BleepingComputer](https://www.bleepingcomputer.com/news/security/ubiquiti-warns-of-new-max-severity-unifi-os-vulnerability/))

### 4. KDDI telecom breach affects over 12 million people
**Why it matters for audit/CISO/VAPT:** The exposure traces to a shared email platform used by five ISPs — a concentration-risk case study for third-party/vendor-risk assessments of outsourced infrastructure.
Japanese telecom giant KDDI disclosed that email addresses and passwords for over 12 million people were exposed after attackers breached an email platform shared by five ISPs in the country. ([BleepingComputer](https://www.bleepingcomputer.com/news/security/japanese-telecom-giant-kddi-says-data-breach-affects-12-million-people/))

### 5. GitHub "verified" commits can be rewritten into new hashes without breaking signatures
**Why it matters for audit/CISO/VAPT:** Undermines a core assumption in software-supply-chain and CI/CD provenance audits — that a signed commit hash is a unique, tamper-evident fingerprint.
New research shows a signed Git commit's hash isn't unique: without the signing key, someone can produce a second commit with the same files, author, and date that still carries a valid signature. Also flagged on this week's SANS Stormcast (see below). ([The Hacker News](https://thehackernews.com/2026/07/github-verified-commits-can-be.html))

### 6. Gitea Docker flaw actively probed 13 days after disclosure
**Why it matters for audit/CISO/VAPT:** A concrete benchmark for patch-SLA audits on CI/CD infrastructure — attackers weaponized this DevOps-platform flaw almost immediately.
Sysdig observed threat actors probing CVE-2026-20896 (CVSS 9.8), a critical Gitea Docker image flaw stemming from misplaced trust in the X-Web header, just 13 days after the patch was disclosed. ([The Hacker News](https://thehackernews.com/2026/07/threat-actors-probe-gitea-docker-flaw.html))

### 7. "HalluSquatting" — AI coding assistants tricked into installing botnet malware
**Why it matters for audit/CISO/VAPT:** Adds an AI-mediated link to the software supply chain; dependency-management audits now need to account for hallucinated package names, not just classic typosquatting.
Researchers show AI coding assistants sometimes suggest plausible but nonexistent packages; the HalluSquatting technique pre-registers those names with botnet malware so developers who trust the suggestion get infected. ([The Hacker News](https://thehackernews.com/2026/07/new-hallusquatting-attack-could-trick.html))
