# Security & Audit Digest — 2026-07-15

## Top items

### 1. Microsoft's record Patch Tuesday: 622 flaws, two zero-days under active attack
**Why it matters for audit/CISO/VAPT:** Two exploited flaws hit core identity and collaboration infrastructure; CISA is separately urging emergency SharePoint hardening — this is an immediate patch-verification and exposure-tracking priority.
Microsoft shipped 622 CVEs (plus 427 in Chromium/Edge) on July 14, roughly triple last month's record, which researchers attribute to AI-accelerated vulnerability discovery. Two are actively exploited: CVE-2026-56155 (AD FS access-control flaw, remediation due 2026-07-28) and CVE-2026-56164 (SharePoint Server missing authentication, due 2026-07-17); CISA also flagged active exploitation of SharePoint CVE-2026-32201 and CVE-2026-45659. ([source](https://thehackernews.com/2026/07/microsoft-patches-record-622-flaws.html))

### 2. SonicWall SMA1000 zero-days (CVSS 10.0) added to CISA KEV
**Why it matters for audit/CISO/VAPT:** Perimeter VPN/SSL appliances are common in scope for VAPT engagements; a CVSS 10.0 SSRF combined with code injection is a full-compromise chain with a hard federal deadline.
CVE-2026-15409 (SSRF) and CVE-2026-15410 (code injection) in SonicWall SMA1000 appliances are being actively exploited; CISA added both to KEV on 2026-07-14 with remediation due 2026-07-17. ([source](https://www.bleepingcomputer.com/news/security/sonicwall-warns-of-sma1000-flaws-exploited-in-zero-day-attacks-patch-now/))

### 3. Progress ShareFile zero-day confirmed, patch now available
**Why it matters for audit/CISO/VAPT:** Closes the loop on last week's emergency shutdown directive — audit should confirm both the shutdown mitigation was applied and the new patch is deployed.
Progress Software confirmed a high-severity zero-day drove the emergency shutdown of internet-facing ShareFile Storage Zone Controllers and has released a fix. ([source](https://www.bleepingcomputer.com/news/security/progress-confirms-sharefile-zero-day-flaw-behind-storage-zone-shutdown/))

### 4. OAuth client ID spoofing used to validate stolen Entra ID credentials
**Why it matters for audit/CISO/VAPT:** The technique evades standard sign-in telemetry since no application name is logged, undermining a common audit control (reviewing failed-login/app telemetry).
At least two threat actors are spoofing OAuth client IDs to enumerate accounts and test stolen credentials against Microsoft Entra ID without triggering normal detections. ([source](https://thehackernews.com/2026/07/oauth-client-id-spoofing-lets-attackers.html))

### 5. xAI's Grok Build CLI uploaded entire Git repos, not just read files
**Why it matters for audit/CISO/VAPT:** Third-party AI coding tools are a growing data-governance blind spot; audit should verify what AI builders in CI/CD pipelines actually transmit versus their stated scope.
Grok Build v0.2.93 uploaded full repositories, including commit history, to a Google Cloud Storage bucket controlled by xAI, beyond the files a task needed — confirmed by a researcher who captured and cloned an upload. ([source](https://thehackernews.com/2026/07/grok-build-uploads-entire-git.html))

### 6. MemGhost: one email can plant persistent false memories in AI agents
**Why it matters for audit/CISO/VAPT:** A new prompt-injection class for AI copilots with inbox access and persistent memory (e.g., SOC assistants) — relevant as AI agents get more autonomy over investigations.
A single crafted email can trick an AI assistant into saving a false fact, hiding the change, and steering later answers, affecting tools with inbox access and memory. ([source](https://thehackernews.com/2026/07/new-memghost-attack-plants-persistent.html))

### 7. RabbitMQ flaws leak OAuth secrets, break tenant isolation
**Why it matters for audit/CISO/VAPT:** Messaging infrastructure often sits behind weaker access-control review than user-facing apps; these flaws let attackers cross tenant boundaries.
Two access-control flaws in RabbitMQ can expose OAuth client secrets and cross-tenant queue metadata, risking takeover of enterprise messaging infrastructure. ([source](https://thehackernews.com/2026/07/rabbitmq-flaws-could-leak-oauth-secrets.html))

### 8. 11 old Microsoft-signed Linux UEFI shims can bypass Secure Boot
**Why it matters for audit/CISO/VAPT:** Undermines a firmware-level trust control assumed intact in most endpoint baselines; relevant to boot-integrity checks in VAPT and audit scope.
Researchers found 11 legacy, still-trusted UEFI shim binaries that can be abused to disable Secure Boot on most UEFI systems. ([source](https://thehackernews.com/2026/07/11-old-microsoft-signed-linux-uefi.html))
